BETMAN Help

What BETMAN is: an evidence-first racing signal and execution platform. It ranks, filters, and passes. It does not promise winners.

1) Quick Start

  1. Select Country and Day, then choose a Meeting in Meeting Workspace.
  2. Click a race card to load race analysis, runner profiles, and Odds vs Model.
  3. Use tabs for Suggested Bets, Multis, Market Movers, Interesting Runners, Bet Plans, and Performance.
  4. Open AI Chat for bespoke race or multi analysis.

2) Core Tags

  • Recommended Bet: best qualified actionable bet after edge, confidence, and contradiction filters.
  • Odds Runner: biggest live overlay that still passes quality controls.
  • EW: each-way profile where place shape is stronger than a straight win-only read.
  • LONG: long-priced runner with real upside. This is now much stricter than before.
  • AI Winner: current top model/AI win probability runner.
  • #1 Model: raw top model runner in the field.
  • Pedigree Advantage GOLD: only shown when a runner has a clear field-relative bloodline edge. No tag is required in every race.

3) Form + Track Tags

  • HOT FORM: last start must be top 3. If latest run misses that gate, the horse cannot be HOT.
  • SOLID FORM: credible recent profile, but not strong enough for HOT.
  • MIXED FORM: conflicting profile or unreliable recent body of work.
  • COLD FORM: repeated weak recent profile. COLD runners are blocked from standard win recommendations.
  • Track labels: Loves This Ground, Suited, Neutral, Query, Dislikes This Ground.

4) Runner Profile updates

  • Sire, Dam, and Dam Sire are now clickable in Runner Profile.
  • Clicking a bloodline opens the Bloodline Library commentary popup with summary, style, strengths, cautions, and trait profile.
  • Pedigree Advantage uses sire, dam, dam sire, female-family scaffold, cross-patterns, and race-archetype priors.

5) Signal, Edge, and Threshold Policy

  • Implied %: market probability from odds (100 / odds).
  • Model %: BETMAN's probability after model + confidence blending.
  • Edge: Model % - Implied %. Positive edge is necessary, but not sufficient.
  • Ped Fit / Ped Conf / Ped Edge: pedigree fit score, pedigree confidence, and field-relative pedigree edge.

Active threshold policy: Main bet = edge ≥ 6 pts and confidence ≥ 65. Secondary bet = edge ≥ 4 pts and confidence ≥ 58. BETMAN now blocks toxic signal buckets like 0–2 pt edge, 2–4 pt edge in 5–8 odds, and the unstable 20–25% model band unless exceptional support exists.

6) PERFORMANCE page

  • BETMAN RETURN: broken out into win-bets and exotics so performance is visible, not blended into one vague number.
  • Pedigree Advantage Performance: tracks pedigree-tagged bets, win rate, ROI, average score, confidence, and edge.
  • Signal Threshold Audit v2: shows good buckets, bad buckets, blocked buckets, and the current threshold policy.

7) AI Chat Modes

  • Race Context: single-race ranked view + risks.
  • 2/3/4-Race Multi: cross-race structures with combined logic.
  • Same-Race Multi: same-race structure analysis.
  • H2H: head-to-head matchup analysis.

8) Account & Users

  • Open Auth to manage credentials.
  • Admins can create/delete users and reset passwords.
  • New user onboarding requires: Name + Email + Password plus verification gate.

9) Troubleshooting

  • If data looks stale, use Refresh.
  • If behavior looks old after updates, hard refresh browser cache.
  • If a tag disappears after a change, check PERFORMANCE and the Signal Threshold Audit first — BETMAN may now be blocking that bucket by policy.
  • If a pedigree tag is missing, that can be correct: there does not need to be a Pedigree Advantage in every race.

Decision standard: Verdict → Evidence → Risk → Action/Pass. If edge is weak, data is noisy, or the bucket is blocked, BETMAN should pass.
