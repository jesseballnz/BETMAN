# AI, Model Tuning & Bakeoff Audit — 26 Mar 2026

## Scope
Complete audit of the AI engine, model tuning parameters, bakeoff evaluation framework, and meeting notes integration.

## Findings & Fixes Applied

### 1. Meeting Notes — Not Reaching AI Models (CRITICAL · FIXED)

**Finding:** User meeting notes (added via the Meeting Intel panel in the UI) were sent from the frontend in the `userNotes` field of the `/api/ask-selection` payload, but the backend **never extracted or injected them** into the AI context. The notes were dead data — the AI had no visibility into them.

Additionally, the `race-analysis` and `strategy` handlers in the frontend did not include `userNotes` at all (only the `chat` handler did).

**Fixes:**
- `buildAiContextSummary()` now extracts `clientContext.userNotes` (up to 5 entries, 120 chars each) and appends a `User meeting notes (Meeting): note1 | note2` line to the AI context.
- Frontend `race-analysis`, `strategy`, and `bakeoff comparison` payloads now include `userNotes`.
- Both system prompts (`BETMAN_ANALYST_SYSTEM_PROMPT` and `BETMAN_CHAT_SYSTEM_PROMPT`) updated with rules 13/14 and 9/10 respectively, instructing the AI to incorporate meeting notes and meeting profile data.

### 2. Meeting Profile Data Loss (MEDIUM · FIXED)

**Finding:** `formatMeetingProfile()` stripped `track_condition` and `rail_position` from the meeting profile when formatting for the AI context, even though this data was available in the raw profile JSON.

**Fix:** `formatMeetingProfile()` now includes `trackCondition` and `railPosition` fields in its output when present.

### 3. AI Instructions — Simulation Language & Punter Panel (MEDIUM · FIXED)

**Finding:** `instructions/instructions.md` contained:
- "Now run 999999 simulated races" — unachievable by an LLM, causes hallucination
- "Make up names" — vague punter panel instruction
- No mention of meeting profile or user notes data

**Fixes:**
- Replaced simulation instruction with probability estimation guidance
- Tightened punter panel instruction to name distinct angles (speed-map analyst, form-line expert, value hunter)
- Added explicit sections for Meeting Profile Bias and User Meeting Notes interpretation
- Updated template headers: "Simulation Model" → "Probability Model"

### 4. System Prompts — Missing Meeting Context Rules (MEDIUM · FIXED)

**Finding:** Neither `BETMAN_ANALYST_SYSTEM_PROMPT` nor `BETMAN_CHAT_SYSTEM_PROMPT` mentioned meeting profiles or user notes, so the AI had no instruction to use this data even when present in context.

**Fix:** Added rules 13–14 to analyst prompt and rules 9–10 to chat prompt for meeting profile bias and user meeting notes.

### 5. AI Chat Window — Buttons Overflowing Panel (LOW · FIXED)

**Finding:** The `.ai-chat-input-row` (input + 3 buttons) used `display:flex` without `flex-wrap:wrap` inside a 360px panel. On narrow viewports, the "Analyze Basket" and "⚖️ Compare" buttons could overflow outside the panel boundary. The panel itself had no `overflow:hidden`.

**Fixes:**
- Added `flex-wrap:wrap` to `.ai-chat-input-row`
- Added `overflow:hidden` to `.ai-chat` panel container

### 6. Bakeoff — Auth Env Var Inconsistency (HIGH · FIXED)

**Finding:** `model_bakeoff.js` used `BETMAN_USERNAME`/`BETMAN_PASSWORD` as fallback env vars, while `model_sanity_test.js` used `BETMAN_USER`/`BETMAN_PASS`. This caused silent auth failures when only one set was configured.

**Fix:** Both scripts now check all three env var patterns: `BAKEOFF_USER`, `BETMAN_USERNAME`, and `BETMAN_USER` (with corresponding password variants).

### 7. Bakeoff — Missing Fetch Timeout in Sanity Test (MEDIUM · FIXED)

**Finding:** `model_sanity_test.js` had no `AbortController` timeout on its fetch calls, unlike `model_bakeoff.js`. A slow/hanging model would block the sanity test indefinitely.

**Fix:** Added 120s `AbortController` timeout to the sanity test fetch.

### 8. Bakeoff — Answer Truncation Too Aggressive (MEDIUM · FIXED)

**Finding:** `model_bakeoff.js` truncated recorded answers to 2000 chars, making post-hoc quality auditing difficult for longer race analyses.

**Fix:** Increased truncation limit to 4000 chars.
